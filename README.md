# Krankenhausverwaltungsdatenbank (Backend-Prototyp)

Dieses Projekt ist ein Backend-Prototyp, der mit Python und PostgreSQL entwickelt wurde.

Es demonstriert relationales Datenbankdesign, Fremdschlüsselbeziehungen, Viele-zu-Viele-Beziehungen, Indizierung und strukturierte SQL-Abfragen mit psycopg2.

## Funktionen

- PostgreSQL-Relationales Schema
- Viele-zu-viele-Beziehung (Patient ↔ Krankheit)
- Fremdschlüsselbeschränkungen mit ON DELETE-Regeln
- Indizierte Join-Spalten für bessere Performance
- Strukturierte Abfrageschicht
- Dockerisierte PostgreSQL-Umgebung

## Verwendete Datenbankkonzepte

- Primärschlüssel & zusammengesetzte Schlüssel
- Fremdschlüsselbeschränkungen
- CASCADE- & SET NULL-Verhalten
- Indexierungsstrategie
- JOIN-Abfragen
- EXPLAIN-fähige Struktur

## Technologie-Stack

- Python
- psycopg2
- PostgreSQL
- Docker

## Zweck

Dieses Projekt diente als Vorbereitung auf ein professionelles Backend-Praktikum mit Schwerpunkt auf PostgreSQL-Architektur und API-Entwicklung.
